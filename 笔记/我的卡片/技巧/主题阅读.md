主题阅读是:从原本的从书本中获取知识,转变为读者从多个作者哪里追问并且构建问题的答案.
# 将主题定义到"可检索"精度
需要将主题变成可以操作的的步骤.
```?
这里如何定义可以检索的精度?
```
例如:
分解问题的主题是:我们如何将一个复杂的问题,分解为可执行的任务,并合并成完整解决路径.
"面对复杂问题如何分解" 改为:
> 1.我如何判断"一个问题拆解到可以执行"的颗粒度?
> 2.如何定义"什么是真正可以执行的下一步任务"?
> 3.如何将这些任务组合成一个可以推进的路径?

# 主题阅读五部分
## 步骤1 选择一个真实问题作为锚点
没有真实的问题,最终一定失败

## 步骤2 为分解阶段,准备一个"固定提问清单"?
主题阅读模板?
```txt
如何判断"问题拆解"拆对了?
如何衡量"任务可以执行了"?
作者是否给出了"停止分解"的标准?
任务之间的顺序是如何确定的?
如何处理不确定性和反馈?
```

## 步骤3 获取信息的时候,只要动作,不要观点
不要获取作者的观点,只获取其中的动作.  
例如:  
```txt
波利亚<<如何解题?>>
如何分解?
在解题之前,写下 已知  + 未知  + 约束

<<搞定>>
什么是任务停止?
如果我无法确定,当前任务是在某个场景下,使用某个工具,执行某个操作,那么这个任务就不算是任务,还得分解
```
只记录:句式 ,动作,判断标准

## 步骤4 横向对齐 将不同书本"概念"放一起进行比对
这是整个 "主题阅读的精华"

|维度|Pólya|GTD|Lean|
|---|---|---|---|
|分解起点|未知/已知|模糊承诺|核心假设|
|任务定义|子问题|下一步行动|验证实验|
|停止标准|可直接求解|可立刻执行|有反馈|
|路径构建|辅助问题链|项目推进|假设验证顺序|
维度这些信息是:可检索的主题阅读中就已经确定的.这里直接拿过来作为维度信息.
```?
这些概念在不同书本中可能有不同的名称
且这些名称的组织方式有有差异?
这个时候哪来对比?其中会有大量的差异?
```

## 步骤5 从主题阅读的中间结果中,构建属于自己的流程
> 把不同书本中重复出现的类似流程,合并成为同一条流程.


例如:
```txt
明确不可变目标
列出当前最大未知
把未知写成假设
为假设设计 可执行可验证动作
任务可执行 -- > 停止分解
对这些动作按照"致命性排序"
```

这一步是 主题阅读的关键

# 在主题阅读中,不同作者的差异
在主题阅读中,由于有不同作者对同一概念的不同表述,会存在大量的差异.其中相同的部分是关键,而差异部分也是信息,而不是噪音.  
对于差异部分需要进行标记+分类+延迟处理.
差异有四类:

----

**一个主题 + 一个真实问题 + 一组固定提问**

用提问审问每本书，  
用表格对齐不同作者，  
提取反复出现的动作作为默认流程，  
把冲突的地方变成“如果-那么”的分支规则，  
最后用自己的语言，写出一套可执行的方法。

---
 一个主题  + 一个真实问题 + 一个固定提问


---
如何获取固定的提问.
问题来源有三个:
- 目标驱动
- 过程驱动
- 失败驱动
问题的套路:
- 通用骨架
	- 目标是什么
	- 什么是成功
	- 什么是结束
	- 最小动作?
	- 如何反馈?
- 主题特殊
	- 不同主题,有不同的实现情况
- 个人情况
	- 不同的人+主题 ==> 问题也有差异
---
为什么是真实的问题
为当前待学习的知识准备一个真实的问题.通过这个问题对当前学的知识进行压力测试,确定当前知识的边界以及有效性.另一通过真实的案例,帮助补全当前知识的情境,细节判断成本因为传播而忽略的隐形知识.让阅读主题更加容易复用和迁移.
