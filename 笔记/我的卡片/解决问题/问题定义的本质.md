# 核心理念
问题不是孤立的症状,而是系统运转过程中的约束点.  
解决问题的过程:  
> 识别症状 --> 选定系统 --> 验证假设  --> 确定问题 --> 迭代优化

# 第一步:识别症状
## 1.1 症状的本质(和系统之间的关系)
症状是系统运转失衡的外在表现
症状不是孤立存在一定存在于一个系统中
症状分为:浅层症状 vs 深层症状

## 1.2 症状识别的要点
频率,强度,趋势
症状之前的因果关系和共生关系
症状对这个系统的约束
## 1.3 症状的关键点
症状 ≠ 问题,症状是通向问题的线索
# 第二步:选择工作系统(核心)
## 2.1 心智的转变
错误认知:不是完全懂了上下文才能确定系统
正确认识:系统不是被发现的,而是被"**选定**"的
这是一个工具性决策,而不是认识论终点.

## 2.2系统选择的判定标准
唯一标准:
> 在整个系统中,我是否有能力 观察,验证,操作其中的某些元素?

如果完全能够 --> 系统小了,考虑扩展
基本能够 --> 刚刚好
完全不能够 --> 系统大了,需要缩小

## 2.3 系统选择的三种资源
### 2.3.1 基本元信息(概念工具箱)
不需要专业知识,只要有常识就能够理解的知识.
举例十个:
- 注意力(稀缺资源)
- 认知负荷(脑子的上限)
- 反馈回路(正循环/负循环)
- 习惯回路(触发-行动-奖励)
- 内在动机(自主,胜任,关联)
- 阻力/助力(变容易/变困难)
- 反馈延迟(行动和结果之间的时间差)
- 意志力(有限的心理力量)
- 环境塑造(环境影响人的行为)
- 系统约束(限制整体表现的瓶颈)
特点:
- 基础(只需要常识就能够理解)
- 有力(能够解释绝大多数现象)
- 可操作(易于识别,测量验证,操作)

### 2.3.2 层级区分能力
能够区分当前系统属于哪一个层级?
- 个人层级(个人特征,能力)
- 行为层级(操作行动,习惯)
- 任务层(任务设计,难度)
- 环境层(物理/环境机制)
- 机制层(奖励,反馈,动机)
- 系统层(多种元素相互作用)
在这里,哪一层更能说得通,就放在哪一层?
### 2.3.3 接收系统会升级的心理预期
系统是会升级的,在行动中会对问题的了解更加精确.这样系统也会越来越好.
```txt
系统V1(粗糙)
- 行动验证
- 发现新元素
系统V2(精细)
- 心动验证
系统V3(进步)
```
这种变化不是之前系统错误了,而是你进步了.

## 2.4系统搭建的实用方法
### 2.4.1 用元信息搭积木
- 1.识别症状中涉及的元信息
	- 这些症状和哪些关键元信息有关
- 2.用元信息搭建工作系统
	- 将元信息组成一个运行框架
	- 标注 元素 ,关系 ,运行机制
- 3.识别可操作的杠杆点
	- 找出系统中哪些元素可以操作
这个过程不求精确,但求能够解释和运行起来

## 2.5 系统定义的三个版本
### 2.5.1 系统定义的三个标准
- 最小可运行系统(MVP):能立即开始的最小系统
- 当前工作系统:正在操作的工作系统
- 潜在扩展系统:未来可能需要考虑的更大系统
保证视野开阔的同时,也保证可操作性

# 第三步:提出并验证假设
## 3.1 假设的性质分类
- 确定事实:有直接证据支持
- 合理推断:有常识推断,待验证
- 知识盲区:需要学习新知识,以及单独探测
## 3.2 假设验证的方法
### 3.2.1 多角度验证
- 事实验证:事实是否成立
- 多角度验证:多个角度查看是否依然成立
- 逻辑检查:因果关系是否成立
- 行动验证:小的行动来验证当前系统是否成立
### 3.2.2 迭代收窄
- 不断修正假设
- 收窄问题的可能空间
- 确定最终的核心问题
## 3.3 快速获取关键上下文的方法
- 症状:持续了多久,频率如何,情境情况
- 边界:目标是什么?还有其他什么影响
- 约束:有哪些不想动的?愿意付出资源有多少?
- 期望:理想状态是什么?

## 3.4 借用参考系统
- 类似案例
- 通用模型
- 基础研究
对于参考不是照抄,而是根据场景进行微调

## 3.5 设定系统锁定期
避免过渡灵活
- 一个系统使用至少两周时间
- 如果失败三次以上,才升级模型
- 记录系统的演进历史.

# 核心原则
## 工具原则
系统是解决问题的工具,而不是真相
选择系统的时候选最有用的,而不是最正确的

## 可操作原则
评判标准:可观测,可验证,可修改
如果不可操作,换系统.

## 渐进性原则
从小系统到大系统,从粗糙到精确
在行动中逐步迭代

## 多方受益原则
好的问题定义,会让系统中的更多要素收益
避免零和博弈

## 容错原则
预留容错空间,不要太激进
系统会在行动过程升级,这是正常情况

## 务实原则
不求完美,只要能够行动就可以(可行动清晰)
30%精确度就可以行动,行动能够提升精确度
