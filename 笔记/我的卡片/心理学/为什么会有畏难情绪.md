# 困境
遇到生活中的绝大多数事情的时候,常常会有畏难情绪,会提不起劲儿来,更多的事情是干了一半以后烂尾了.
遇到这种问题的原因是:对于当前的事情的判断`不清晰,不可预测,不可控`.
内心中:
- 不清晰的知道事情的细节和完成方法
- 不确定任务当中的风险以及汇报
- 无法确定努力是否有效的完成最终结果
- 也没有办法定位当前的状态,以及需要踏出的下一步
由于人的自我保护机制,遇到一个情况的时候会提问:
- 我在干什么?
- 接下来会发生什么?
- 有什么风险?
当这些问题都不确定的时候,这个时候大脑是不会行动的,而是会触发撤离的情绪.

# 终点
对问题的结构化是动力的前置条件,而不是结果.不是因为有了动力,才能对现状进行结构化.而是对现状进行了结构化,搞清楚了现状,让问题变得可控了,这个时候才会有动力.结构的结果是:
- 对问题和现状有了解,知道自己在干什么?(清晰)
- 知道下一步要做什么,也知道下一步完成后的结果是什么?让问题变得可以预测.(可以预测)
- 也知道这些问题的推进自己是有能力解决的,一步步向前推进的(可控)
# 为什么会畏难
在没有搞清楚问题的模型的时候,
```txt
行动 --> 盲目 --> 高风险 --> 回避
```

# 降低问题的维度
遇到问题的时候,不是立刻进行结构化,而是降低问题维度.
## 没有清晰目标
- 知道要干什么,但是说不清楚
- 知道目标重要,但是无法量化
由于不知道具体要干什么也不知道到底有多重要.无法反推当前的路径.
## 问题跨度太大
- 有一个抽象的目标
- 有具体的实现细节
但是这种抽象的目标和具体的实现细节之间无法关联上.导致了最终的失败.
## 成功条件不清晰
- 不知道什么时候停止
- 不知道什么是好的,什么是不好的
不知道什么是第一步,容易无限衍生

# 思维转变
第一步不是开始行动,而是构建一个"可以解决问题的模型".
整体解决问题的步骤是:
- 生成一个问题解决方案
- 找出第一步
- 执行
- 反馈
- 修改问题解决方案
# 解决方案
三个问题实现降低维度.
- 当前问题的"最小可描述版本"
用20个字,描述当前问题是解决一个什么东西.
画出当前系统的方案
- 如果只干30分钟,我能解决其中的
不是解决问题,而是解决当前问题模型中的一个状态.
找出当前问题中的所有的存在的用户.
- 什么时候算结束
必须可以观察,可以衡量的
不管对不对,列出所有可能的用户状态.

# 当前问题
不是畏难,而是拒绝在一个没有结构化问题的情况下采取行动.
这是一种搞认知头脑的副作用.眼高手低.
解决方案是:每次行事之前第一步是对问题进行结构化.


# 大脑的反应
对于当前问题的不清晰,不可预测不可控,会导致大脑发出撤退的反应.
- 不清晰,会导致大脑中需要关注多个信息,从而导致负荷过大.这样会消耗大脑的情况.
- 不清晰,无法根据a情况到b中间路径的清晰认识,会让大脑认为这个任务是不可完成的.从而确定是撤退
- 不可预测会导致奖励系统失效,可以预测的结果的完成会导致多巴胺的分泌,触发奖励系统.没有可以预测的结果,就会导致新的动力的产生.这种情况就会导致毫无动力
- 不可控会导致原始的安全失控反应,对于不可控的场面,大脑会使用拖延和后退的机制.
# 结构化的步骤
- 拆解任务:将大问题拆解成小的步骤.
- 明确标准:确定做好和做完的标准.
- 识别变量:哪些是可控的,哪些是不可控的
- 设置反馈:设置检查点来验证当前的进展.
# 思考的方式
- 当前可以最开始的5分钟作为的任务有哪些?
- 这个事情可控的和不可控的部分在哪里?
- 怎么算是有进展?
- 这个事情最差的情况是什么?
